This was developed using Angular CLI 12
Angular gets the values (pokemon/:id) from the path in the Url using this.route.paramMap.subscribe((params) => { })

The Param is included in the Url that does look like a path. This can be used as a reference link to be shared accross different party. This should contain non-sensitive data. It can be used in GET requests. And, in most cases, when referring to a specific entity (i.e. id), it is usually used with POST, PUT, PATCH and DELETE requests

In Angular routing, you need to specify the path for the param like in the below syntax.


GET /param/pokemon // gets all data in the list
GET /param/pokemon/:id // gets a specific data in the list
